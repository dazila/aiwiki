# Решения по проекту

Журнал ключевых архитектурных и инструментальных решений с обоснованиями.

## Хост-инфраструктура

**Решение:** ПК на Ubuntu Server 24.04 LTS как единственный гипервизор для Light/Normal. Yandex Cloud — только для Hard.

**Альтернативы:** Локально на маке (Vagrant + Parallels), гибрид мак+облако.

**Обоснование:** ПК — x86_64, 128 ГБ RAM, 32 потока. Все Vagrant-боксы и Ansible-роли проверены на x86. Мак на Apple Silicon — постоянные мелкие несовместимости, плюс 16 ГБ RAM впритык под 4 ВМ + Ollama. Облако оставляем под Hard, где нужен managed Kubernetes.

## Сеть

**Решение:** Bridged-сеть в `192.168.1.0/24`, ВМ получают IP в локалке, MAC фиксируется в `Vagrantfile` и резервируется на роутере.

**Альтернативы:** NAT с пробросом портов, libvirt host-only сеть `192.168.122.0/24`.

**Обоснование:** Bridged — самый прямой путь: с любого устройства локалки видно всё, без проброса портов и без специальных адресов хоста. mDNS через Avahi даёт читаемые имена `aiwiki-*.local`.

## LLM-модель

**Решение:** Стартовая — `qwen2.5:3b-instruct`. Запускается на CPU (E5-2683 v4, 32 потока).

**Альтернативы:** `llama3.2:3b-instruct`, `gemma2:2b`, `qwen2.5:7b-instruct`, `qwen2.5:1.5b`.

**Обоснование:** qwen2.5 — лучший русский язык и tool-calling в этой весовой категории. 3B — компромисс между качеством и скоростью на CPU. Если будет невыносимо медленно — даунгрейд на `qwen2.5:1.5b` или апгрейд через GPU (RTX 3060 12GB).

## GPU

**Решение:** Стартуем без GPU. Если упрёмся в производительность Ollama — RTX 3060 12 ГБ б/у (25–30к).

**Обоснование:** Для проверки логики Light/Normal производительности CPU достаточно. GPU имеет смысл, когда n8n начнёт массово вызывать модель из workflow (Hard, parallel workers).

## Ansible / Vagrant

**Решение:** Vagrant и Ansible живут на ПК-хосте, не на маке.

**Обоснование:** libvirt — локальный для ПК, нет смысла гонять remote-провайдеры. Управляющая машина внутри инфры — это нормальная практика, заодно учебный навык.

## Репозиторий

**Решение:** Публичный репозиторий `github.com/dazila/aiwiki`.
