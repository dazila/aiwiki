## Сеть и виртуализация

**bridge (Linux bridge, мост)** — программный коммутатор уровня L2 в ядре Linux. Передаёт фреймы между подключёнными интерфейсами по таблице MAC-адресов. У нас — `br0` поверх физического `enp6s0`, к нему цепляются виртуальные интерфейсы ВМ.

**mDNS (Multicast DNS)** — протокол для разрешения имён в локальной сети без центрального DNS-сервера. Использует мультикаст-группу `224.0.0.251` UDP/5353, домен `.local`. См. RFC 6762.

**DNS-SD (DNS Service Discovery)** — поверх mDNS, для обнаружения сервисов: `_ssh._tcp.local`, `_http._tcp.local` и т.п. См. RFC 6763.

**Avahi** — реализация mDNS/DNS-SD для Linux. Демон, который анонсирует `<hostname>.local` и слушает запросы.

**libvirt** — демон + API над разными гипервизорами (QEMU/KVM, Xen). Унифицирует управление ВМ, сетями, томами.

**KVM** — модуль ядра Linux, использует Intel VT-x / AMD-V для аппаратной виртуализации. Без него QEMU работает в режиме программной эмуляции (медленно).

**vagrant-libvirt** — плагин Vagrant, говорит на API libvirt.

**bridged network (в libvirt)** — режим, в котором виртуальный интерфейс ВМ цепляется к физическому мосту хоста. ВМ видна в локалке как обычное устройство.

**management network (в vagrant-libvirt)** — отдельный мост `virbr1` в подсети `192.168.121.0/24`, через который Vagrant ходит в гостя для SSH и provisioning.

## Прокси и безопасность

**SOCKS5** — протокол прокси-сервера на 5-м уровне OSI. Поддерживает любой TCP/UDP-трафик. Не понимает протокол приложения.

**Privoxy** — лёгкий HTTP-прокси, в нашем сетапе используется для конвертации SOCKS5 → HTTP-proxy (потому что многие CLI-инструменты не умеют SOCKS).
