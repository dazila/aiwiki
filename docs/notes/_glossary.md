## Сеть и виртуализация

**bridge (Linux bridge, мост)** — программный коммутатор уровня L2 в ядре Linux. Передаёт фреймы между подключёнными интерфейсами по таблице MAC-адресов. У нас — `br0` поверх физического `enp6s0`, к нему цепляются виртуальные интерфейсы ВМ.

**mDNS (Multicast DNS)** — протокол для разрешения имён в локальной сети без центрального DNS-сервера. Использует мультикаст-группу `224.0.0.251` UDP/5353, домен `.local`. См. RFC 6762.

**DNS-SD (DNS Service Discovery)** — поверх mDNS, для обнаружения сервисов: `_ssh._tcp.local`, `_http._tcp.local` и т.п. См. RFC 6763.

**Avahi** — реализация mDNS/DNS-SD для Linux. Демон, который анонсирует `<hostname>.local` и слушает запросы.

**libvirt** — демон + API над разными гипервизорами (QEMU/KVM, Xen). Унифицирует управление ВМ, сетями, томами.

**KVM** — модуль ядра Linux, использует Intel VT-x / AMD-V для аппаратной виртуализации. Без него QEMU работает в режиме программной эмуляции (медленно).

**vagrant-libvirt** — плагин Vagrant, говорит на API libvirt.

**bridged network (в libvirt)** — режим, в котором виртуальный интерфейс ВМ цепляется к физическому мосту хоста. ВМ видна в локалке как обычное устройство.

**management network (в vagrant-libvirt)** — отдельный мост `virbr1` в подсети `192.168.121.0/24`, через который Vagrant ходит в гостя для SSH и provisioning.

## PostgreSQL

**cluster** - сервер для одной или нескольких баз данных (примерно равно instance).

**schema** - аналог неймпейса - логическое разделение таблиц в postgres

**database** - база данных, совокупность объектов (таблиц), объединенных в схемы

**role** - роль в postgres. С атрибутом LOGIN работает как пользователь, без - работает как группа для назначения прав.

**user** - это роль с правами LOGIN для подключения к базе

**owner** - владелец объекта в pg, обладает полными правами на объектом

**PUBLIC (role)** - псевдо-роль в pg, объединяющая в себе все роли

**PUBLIC (schema)** - дефолтная схема, которая есть в любой базе данных. До PG15 имела GRANT CREATE to PUBLIC, после это убрали

**pg_hba.conf** - конфиг-файл host-based authentication с правилами аутентификации при подключении к базам

**postgresql.conf** - основной файл конфига pg. В числе прочего содержит настройки listen_addresses, port и т.д.

**listen_addresses** - параметр из postgresql.conf, указывает на каком ip-адресе pg будет слушать подключения (можно указать несколько адресов через запятую или * - для любых адресов)

**search_path** - упорядоченный перечень схем, по которым pg будет производить поиск в случае указания неполного имени объекта

**trust** - один из методов аутентификации пользователя из pg_hba.conf. Обозначает, что подключения отрыты всем без аутентификации

**scram-sha-256** - метод аутентификации из pg_hba.conf, учетные данные хэшируются, при этом хэш, который хранится в pg_authid не равен тому, что будет отправляться клиентом серверу

**md5** - устаревший метод аутентификации из pg_hba.conf. Подвержен replay-атакам

**peer** - метод аутентификации из pg_hba.conf для подключения по unix-socket

**GRANT** - назначение прав на объект

**REVOKE** - удаление прав с объекта

**DEFAULT PRIVILEGES** - права по умолчанию для вновь создаваемых объектов

**idempotent SQL** - свойство команды или перечня команд SQL, применение которых повторна на том же наборе данных, на котором до этого эта команда или скрипт была выполнены, не вносит никаких изменений (действия не выполняются повторно)

**DO block** - анонимный блок команд PL/pgSQL, позволяет использовать IF/LOOP и т.п. без создания функции и выполняющийся как единая транзакция

**\gexec** - мета-команда psql, позволяющая отправлять на выполнение строку(или строки) полученные от первой команды SQL. Позволяет делать идемпотентной работу с командами, которые не выполняются как транзакции, вроде создания баз

**\i** - мета-команда psql, позволяющий выполнить sql-скрипт из файла

**SIGHUP** - сигнал в linux, который позволяет дать процессу команду на повторное применение конфига (номер 1)

**reload** - команда systemctl (systemd) для отправки процессу сигнала SIGHUP (повторное применение конфига)

**restart** - команда systemctl (systemd) для отправки процессу сигнала SIGTERM (и SIGKILL после таймаута) и повторного запуска процесса

**pg_isready** - утилита для проверки готовности postgresql принимать подключения

**psql** - утилита для работы с postgresql

**PGDATA** - директория с данными баз в postgresql

**WAL** - write-ahead logs - логи транзакций в postgresql

**pg_dump** - утилита для снятия логического дампа базы postgresql

**pg_basebackup** - утилита для побитового копирования базы в postgresql

## Прокси и безопасность

**SOCKS5** — протокол прокси-сервера на 5-м уровне OSI. Поддерживает любой TCP/UDP-трафик. Не понимает протокол приложения.

**Privoxy** — лёгкий HTTP-прокси, в нашем сетапе используется для конвертации SOCKS5 → HTTP-proxy (потому что многие CLI-инструменты не умеют SOCKS).
